Introduction

Total paralysis patients often rely on eye-based communication to relay basic needs. However, the success of communication hinges on the receiver's ability to interpret the sender's intention.

Therefore, we propose an attention-based CNN model to detect, classify, and translate eye movements into common phrases that can reach broader audiences. These classifications (left, right, up, and blink) can be integrated into three-component sets to key into a predefined eye alphabet that conveys information.

Preprocessing

We used unlabelled facial data from the Closed Eyes in the Wild dataset and manually labelled each image.

The data was then shuffled and split at a 3:1 ratio between training and testing. We normalized each image by scaling pixel values between 0 and 1. Each label was one-hot encoded.

To manage imbalance in the dataset in one version of the model, we downsampled “blink” images and augmented all others (e.g. crop, flip, contrast, saturation).

Model Architecture

Our model architecture is a supervised, attention-based CNN comprised of four convolutional layers with LeakyReLU activations, each followed by a max-pooling operation incorporated with an attention mechanism.

These attention layers are concatenated into a feature vector that is fed through three fully-connected dense layers before getting passed to a softmax activation function to return the probabilities across the four classes.

Challenges

Imbalanced Dataset: Our model was learning to "cheat" by only predicting the dominant class (blink). To mitigate this, we experimented with weighted loss, weighted sampling, data augmentation, and downsampling.

Overfitting: Due to the limited variation within our small dataset, the complex architecture of our model was prone to capturing noise. We trialed regularization techniques and dropout layers to temper the model complexity.

Reflection

Trade-Offs: All architectural choices introduced trade-offs. Model design rarely focuses on maximizing a single metric, but rather balancing interpretability, robustness, and performance.

Mitigating Bias: Addressing imbalance in the dataset was crucial to ensure that our model could effectively classify all categories. Acknowledging this bias is not only a technical need, but also an ethical responsibility. For assistive technologies in particular, these types of failures disproportionately affect already vulnerable users.
